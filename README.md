# meal-planner

适合小基数人群的本地化减脂餐计划 H5。

Vue 3 + Vite 单页应用，数据只保存在当前浏览器本地，无账号、无云同步。

**部署方式:** GitHub Pages

---

## v0.2.4

### 已具备功能

- **身体资料输入：** 性别、年龄、身高、体重、目标体重、日常活动水平、运动情况
- **作息与饮食场景：** 起床/睡觉时间、早餐习惯、午餐/晚餐结构
- **小基数推荐方案：** 进食方式（正常三餐 / 三餐+加餐 / 14:10 进食窗口 / 16:8 进食窗口）、热量缺口建议、宏量营养素分配
- **进食窗口：** 14:10 / 16:8 支持按起床、睡觉和早餐习惯推导默认窗口，确认餐次时可编辑窗口开始时间
- **餐次时间：** 确认餐次时可直接编辑每餐时间，进食窗口方案会校验所有餐次位于窗口内，并可一键按窗口自动分配
- **热量缺口：** 10% / 15% / 20% 可选，小基数保护
- **宏量营养素：** 蛋白按体重 g/kg 计算、脂肪最低 0.8g/kg、碳水用剩余热量，含安全兜底
- **个人可用食材池：** 内置默认食材，支持勾选/取消、自定义食材、删改
- **自炊餐型模板 / 真实餐点：** 早餐、午餐、晚餐、加餐，含合理餐型分配
- **食谱操作：** 全量刷新、单餐替换、锁定餐点
- **餐点编辑：** 支持修改单餐食材和克重，保存后名称、食材、热量和宏量营养同步更新
- **当天食材编辑：** 可为某一天单独选择食材池，并只刷新当天未锁定/未编辑餐点
- **数据备份：** 我的页支持导出/导入本地 JSON 备份
- **体重记录：** 新增/删除，含饥饿感、执行度、备注
- **趋势视图：** 7 日均重、SVG 趋势图、动态建议
- **今日打卡：** 餐单完成度、外食/运动、睡眠、饥饿感、执行度 1-5、备注
- **7 天复盘：** 最近 7 天执行摘要、复盘建议、最近打卡记录
- **App Shell：** 生成计划后进入底部 Tab 界面：餐单 / 食材 / 进度 / 打卡 / 我的

### v0.2.4 更新

- 推荐页新增返回资料填写入口，返回后保留已填资料，重新提交会生成新的推荐
- 食材页和当天食材编辑页新增“搜索食材”，支持按名称、分类和标签实时过滤，并保留已选状态
- 页面、编辑器和底部 Tab 切换后会重置到顶部，避免 H5 保留上一页滚动位置

### v0.2.3 更新

- 推荐页将 14:10 / 16:8 明确标记为“进食窗口”，折叠状态仍显示当前方案说明
- 餐次确认页新增顶部返回推荐、可编辑餐次时间、可编辑进食窗口开始时间和窗口校验
- 计划生成、刷新、单日刷新和单餐更换会沿用已保存的 `eatingWindow` 与餐次时间
- 餐单摘要展示实际保存的进食方案和窗口时间

### 隐私说明

- 所有数据只保存在当前浏览器的 IndexedDB 和 localStorage 中
- 换设备不会自动同步
- 清空浏览器数据将丢失所有信息
- 无任何数据发送到服务器

### 已知限制

- 无账号系统
- 无云同步
- 导入备份会覆盖当前浏览器本地数据，请先确认文件来源
- 食材营养数据为产品级估算，非精准实验室数据
- 非医疗建议，不构成健康或营养诊断
